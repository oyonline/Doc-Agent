# V0.2 · 页面任务书

## 文档目的

本文档是 V0.2 实现 UI 的入口指令。代码 Agent 实现每个页面时,以本文档对应的页面任务书为准。

本文档不重新定义状态机(见 `docs/03`)、不定义调度时序(见 `docs/04`)、不定义具体数据值(见 `docs/06`)、不定义会话管理(见 `docs/05`)。每个页面任务书的"数据来源"小节会精确指向应该取哪一份的哪一节。

## 页面清单与实现优先级

| 页面 | 类型 | V0.2.0 | V0.2.1 | V0.2.2 | 优先级 |
|---|---|---|---|---|---|
| 需求工作台 | 新增 | 完整 | — | — | P0 |
| 需求详情页 | 新增 | 完整 | 增强(虚线契约) | 增强(决策与上下文) | P0 |
| 任务板 | 重做 | 完整 | — | — | P0 |
| 任务详情页 | 保留 + 升级 | 完整 | — | 增强(Tab 升级) | P1 |
| 执行详情页 | 保留 | 沿用 V0.1.1 | — | — | P2 |
| 单任务审查包页 | 保留 | 完整(沿用 V0.1.1) | — | — | P1 |
| 需求级审查包页 | 新增 | 完整 | — | — | P0 |
| 右侧状态栏 | 重做 | 完整 | — | 增强(会话信息) | P0 |
| 工作流 / 智能体 / 设置 | 占位 | 占位 | — | — | P3 |

## 页面通用规范

- 沿用 V0.1.1 设计令牌(详见 `app/DESIGN_TOKENS.md`,V0.2 实现时不引入新令牌)。
- 全局外壳沿用 V0.1.1(顶栏 TopBar / 左侧导航 SideNav / 右侧状态栏 RightStatus / 底部事件流 BottomEvents)。
- 中文字体栈;文案统一中文,不夹杂英文。
- 默认严格安全模式(顶栏只展示"严格模式"小标签,详细安全信息在右侧状态栏)。
- 状态色沿用 V0.1.1 设计令牌(成功 / 执行中 / 验证中 / 待审查 / 警告 / 失败 / 待处理)+ 新增需求状态色(待澄清 / 澄清中 / PRD 已定 / 拆解中 等)。

## 各页面任务书

### 需求工作台

#### 页面定位

V0.2 主入口页,代替 V0.1.1 的首页。用户登录后默认落地此页,可看到"我的全部需求"概览以及"系统等我做的决策"。

#### 页面区域

- **区域 1 · 需求列表(主区域,左 + 中)**:展示用户全部需求,按"近期更新"排序。每行显示:需求编号 / 名称 / 主状态 / 异常标签 / 提交时间 / 当前阶段简述。
- **区域 2 · 当前焦点需求(右上)**:固定展示主流程需求-001 的精简卡片,含状态轨迹时间轴 + 任务图缩略图 + "进入详情"按钮。
- **区域 3 · 待你决策事项(右下)**:列出当前需要用户操作的事项,每条显示:事项类型(任务待审查 / 需求待审查 / 澄清等待回复)+ 来源需求 / 任务编号 + "去处理"按钮。

#### 关键交互

- 点击需求列表行 → 跳转需求详情页(来源:`docs/03 §跳转矩阵 - 需求工作台`)。
- 点击"待你决策"事项 → 根据事项类型路由到对应详情页(需求详情 / 任务详情 / 单任务审查包 / 需求级审查包)(来源:`docs/03 §跳转矩阵 - 需求工作台`)。
- 顶栏点击"创建需求" → 唤起新建需求弹层,提交后回到本页 + 新需求出现在列表顶部(来源:`docs/03 §跳转矩阵 - 顶栏`)。
- 弹层提交需求 → 主状态 = 待澄清,产品智能体准备启动(来源:`docs/03 §按钮触发矩阵 - 提交需求`)。

#### 数据来源

- 需求列表数据:`docs/06 §端到端样例需求总览 - 需求清单`(4 个需求样例)。
- 当前焦点需求字段:`docs/06 §主流程需求基本字段(需求-001)`。
- 待你决策事项:`docs/06 §11.5 各时刻"待你决策事项"快照`(根据当前演示时刻取值)。

#### V0.2.0 / V0.2.1 / V0.2.2 差异

- **V0.2.0**:展示 4 个需求样例;待你决策事项最多 3 条(任务-1 待审 / 任务-6 待审 / 需求级待审,根据演示时刻不同显示其中之一)。
- **V0.2.1**:相对 V0.2.0 增加任务-7 待审决策事项。
- **V0.2.2**:无 UI 差异(会话信息只在任务详情页内显示,不在需求工作台显示)。

#### 严禁

- 严禁让需求列表展示伪造的进度百分比。
- 严禁让"待你决策"区域为空(主流程演示中至少 1 条;若实际无,显示"暂无待你决策的事项")。
- 严禁把"创建需求"按钮做成首页 textarea 输入框(违反 docs/00 §V0.2 不做什么)。

### 需求详情页

#### 页面定位

需求级别的"工作台",承载需求从澄清到审查的全生命周期。根据需求当前状态,页面展示不同子视图(单页内切换,不跳转)。

#### 页面区域

页面顶部固定展示需求基本信息(编号、名称、主状态、异常标签、提交时间)。下方主区域根据当前主状态切换视图:

- **视图 A · 多轮澄清**(主状态 = 澄清中):产品智能体提问与用户回答的对话流(3 轮固定演示)+ "提交回答"按钮 + "确认进入 PRD"按钮(第 3 轮后显示)。
- **视图 B · PRD**(主状态 = PRD 已定 / 拆解中 / 执行中 / 待审查 / 已完成):展示 PRD 全文(背景 / 目标 / 范围 / 用户故事 / 需求级验收标准),可折叠展开。
- **视图 C · 可行性评估**(主状态 = PRD 已定及之后):展示风险清单 + 实现方向 + 关键技术决策。已驳回时同时展示二次澄清记录与二次评估对比。
- **视图 D · 任务图**(主状态 = 拆解中 / 执行中 / 待审查 / 已完成):自写 SVG 节点 + 连线展示 7 任务依赖图,任务图节点点击下钻到任务详情页。
- **视图 E · 执行进度**(主状态 = 执行中 / 待审查):任务汇总状态卡片(可与视图 D 任务图并列展示),含调度记录 + 当前活跃任务高亮。
- **底部按钮区**:根据当前状态显示:"启动执行"(拆解中)/ "查看需求级审查包"(待审查)等。

#### 关键交互

- 多轮澄清"提交回答" → 同页刷新(主状态 = 澄清中 · 第 N 轮 → 第 N+1 轮)(来源:`docs/03 §跳转矩阵 - 需求详情页`)。
- "确认进入 PRD" → 同页切换到视图 B(主状态:澄清中 → PRD 已定)(来源:`docs/03`)。
- "启动执行" → 同页切换到视图 E,任务图开始动起来(主状态:拆解中 → 执行中)(来源:`docs/03`)。
- 任务图节点点击 → 跳转任务详情页(来源:`docs/03 §跳转矩阵`)。
- "查看需求级审查包"(主状态 = 待审查时显示) → 跳转需求级审查包页(来源:`docs/03`)。
- 已驳回情形:架构师评估完成后产品智能体自动唤起二次澄清面板,用户回答后自动进入二次评估,无需用户额外按钮(来源:`docs/03 §需求状态机 - 异常标签 - 已驳回` + `docs/06 §可行性评估`)。

#### 数据来源

- 多轮澄清对话内容:`docs/06 §多轮澄清记录`(3 轮固定文案)。
- PRD 全文:`docs/06 §PRD`(6 条需求级验收标准)。
- 风险清单与实现方向:`docs/06 §可行性评估`(5 条风险 + 二次评估降级动线)。
- 任务图节点数据:`docs/06 §任务图与依赖`(V0.2.0 实线版 / V0.2.1 虚线版)+ `docs/06 §节点字段映射`。
- 执行进度数据:`docs/04 §V0.2.1 完整调度时序剧本`(实时映射当前 T 时刻)+ `docs/06 §11.6 状态栏快照`。

#### V0.2.0 / V0.2.1 / V0.2.2 差异

- **V0.2.0**:任务图视图只显示实线箭头,任务-7 节点标注"待 V0.2.1 激活";已驳回动线展示完整(二次澄清 + 二次评估)。
- **V0.2.1**:任务图视图新增虚线契约依赖(任务-2 ↔ 任务-4、任务-3 ↔ 任务-5);任务-7 激活,加入调度链路。
- **V0.2.2**:任务详情页 Tab 升级影响下钻视图,但需求详情页本身无 UI 差异。

#### 严禁

- 严禁把多轮澄清交互做成独立页面(必须嵌入需求详情页)。
- 严禁把视图 A~E 拆成多个独立路由(必须单页内切换)。
- 严禁让任务图节点跳转到任务卡而不是任务详情页(任务图节点 ≠ 任务卡,详见 `docs/03 §任务表示形态说明`)。
- 严禁让 V0.2.0 阶段任务图出现虚线(违反分批纪律)。

### 任务板

#### 页面定位

V0.2 重做的任务总览视图,从"五列状态看板"扩展为"按需求分组的任务池视图"。承载多需求并存场景下的全局任务概览。

#### 页面区域

- **顶部 · 需求过滤器**:可按需求筛选(默认全部需求展开),需求-002 已完成默认折叠。
- **主区域 · 按需求分组的任务卡列表**:每个需求一个分组,分组标题为需求编号 + 名称 + 主状态 + 任务进度(X/N)。组内展示该需求下的全部任务卡。
  - 需求-001 分组:7 任务(V0.2.0 中任务-7 显示"占位 · 待 V0.2.1 激活"水印,不可点击)。
  - 需求-002 分组:已完成需求(本文档不展开任务样例,只占位显示"X 个任务,已结束")。
  - 需求-003、004:无任务(待澄清 / 待审查阶段不展开任务图,分组提示"未进入拆解"或"已完成执行")。
- **任务卡字段**:任务编号 / 名称 / 智能体类型 / 主状态 / 异常标签 / 验证结果(X/N)/ 风险等级。

#### 关键交互

- 点击需求分组标题 → 跳转需求详情页(来源:`docs/03 §跳转矩阵 - 任务板`)。
- 点击任务卡 → 跳转任务详情页(来源:`docs/03`)。
- 点击任务卡"查看审查包"按钮(待审查列任务) → 跳转单任务审查包页(来源:`docs/03`)。
- 任务卡所有字段直接展示,无悬浮态、无展开态(沿用 V0.1.1 任务卡片规范)。

#### 数据来源

- 需求分组数据:`docs/06 §端到端样例需求总览 - 需求清单`。
- 任务卡字段值:`docs/06 §7 任务的样例数据`(各任务"任务卡字段"小节)。

#### V0.2.0 / V0.2.1 / V0.2.2 差异

- **V0.2.0**:任务-7 显示"占位 · 待 V0.2.1 激活"水印,不可点击,字段值为空。
- **V0.2.1**:任务-7 激活,任务卡字段切换为"已完成 / 4/4 / 中风险"。
- **V0.2.2**:任务卡字段不变,**不显示**会话信息(会话信息归任务详情页)。

#### 严禁

- 严禁把任务卡片展示伪造的 token 百分比(违反 `docs/00 §V0.2 不做什么` + `docs/05 §任务卡片不变`)。
- 严禁让任务图节点出现在任务板上(任务图节点只能在需求详情页内)。
- 严禁让任务-7 在 V0.2.0 阶段被当作真实任务展示(违反 `docs/03 §严禁事项`)。

### 单任务审查包页

#### 页面定位

任务级最终决策入口,V0.1.1 资产沿用。从任务板"查看审查包"或任务详情页"查看审查包"或执行详情页"查看审查包"进入。

#### 页面区域

- **任务回顾**(任务名 / 智能体总结)
- **验收标准结果**(N/M 通过)
- **修改文件清单**(可点击行)
- **同页静态差异预览**(默认选中第一个文件)
- **验证结果**(分项展示)
- **风险提示**
- **审查建议**
- **底部固定决策区**:
  - 左侧:`拒绝并反馈`
  - 右侧:`接受结果`(高风险任务需二次确认)
  - 中间或次级:`更多操作` 内含 `重新执行`、`让审查智能体复查`

#### 关键交互

- 修改文件清单行点击 → 同页差异预览刷新(不跳转)(来源:`docs/03 §跳转矩阵`)。
- 点击 `接受结果` → 跳转任务详情页 + 任务待审查 → 已完成 + 进入下游调度(来源:`docs/03 §按钮触发矩阵`)。

### 需求级审查包页

#### 页面定位

需求级最终决策入口,V0.2 新增。所有任务完成后,审查智能体生成跨任务一致性问题清单,本页承载需求级最终人工决策。

#### 页面区域

- **需求回顾**(需求编号、名称、提交时间、完成时间、总耗时)
- **PRD 对比交付**(PRD 6 条验收标准 ↔ 实际交付情况对照表)
- **任务汇总**(7 任务状态 / 验证 / 风险表)
- **跨任务一致性问题清单**(6 条:1 严重 / 2 警告 / 3 提示,可点击进入对应任务详情页)
- **风险评估**(高 / 中 / 低汇总 + 跨任务严重问题计数)
- **审查建议**(建议方案 + 替代方案对照)
- **底部固定决策区**:
  - 左侧:`拒绝并反馈`
  - 中间:`部分接受`(进入选择模式)
  - 右侧:`接受需求结果`(高风险或有严重跨任务问题时需二次确认)
  - 次级:`让审查智能体复查`

#### 关键交互

- 点击任务汇总行 / 跨任务问题清单条目 → 跳转单任务审查包页(下钻)(来源:`docs/03 §跳转矩阵`)。
- 点击 `接受需求结果` → 跳转任务板(或需求工作台) + 需求待审查 → 已完成(高风险二次确认)(来源:`docs/03 §按钮触发矩阵`)。
- 点击 `拒绝并反馈` → 必填反馈 → 跳转需求详情页 + 需求待审查 → 拆解中或执行中 + 全量拒绝并反馈(来源:`docs/03`)。
- 点击 `部分接受` → 进入选择模式(不跳转) → 选择哪些任务保留 / 哪些重做 → 需求待审查 → 执行中 + 部分拒绝(来源:`docs/03`)。
- 点击 `让审查智能体复查` → 主状态保持待审查 + 审查智能体状态变工作中(来源:`docs/03`)。

#### 数据来源

- 需求回顾 / PRD 对比 / 任务汇总 / 跨任务一致性 / 风险 / 审查建议 / 决策选项:`docs/06 §需求级审查包`(10.1~10.7 全节)。

#### V0.2.0 / V0.2.1 / V0.2.2 差异

- **V0.2.0**:任务汇总展示 6 任务(任务-7 占位,标注"待 V0.2.1 激活")。
- **V0.2.1**:任务汇总展示 7 任务(任务-7 激活)。
- **V0.2.2**:无 UI 差异(决策选项与跨任务问题清单不变)。

#### 严禁

- 严禁让需求级审查包页跳过跨任务一致性问题清单(它是 V0.2 核心展示)。
- 严禁让"部分接受"跳转到独立选择页面(必须同页进入选择模式)。
- 严禁让"接受需求结果"在有严重跨任务问题时不触发二次确认(违反 `docs/06 §10.7`)。

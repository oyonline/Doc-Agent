# Gas Town 借鉴点

本文件是历史参考资料，不是当前事实源。

当前唯一事实源见：

- `docs/01-单一事实源-V0.1.1.md`

如果本文件与 V0.1.1 整合版冲突，以整合版为准。

## 一句话理解

Gas Town 是一个面向 Claude Code、GitHub Copilot、Codex、Gemini 等 AI 编码智能体的多智能体编排系统，核心解决多智能体并发工作时上下文丢失、协调混乱、任务追踪不可靠的问题。

## 值得借鉴的点

V0.1.1 只吸收以下产品思想：

- 状态外化。
- 任务优先。
- 输入和输出都可审查。
- 执行过程可追踪。
- 审查前需要验证。
- 人类负责最终决策。
- 成功经验未来可以沉淀为工作流。

## 对 V0.1.1 的采用方式

| 借鉴点 | V0.1.1 采用方式 | 后续版本锚点 |
|---|---|---|
| 工作状态持久化 | 静态任务状态、任务详情页、执行详情页 | 真实状态恢复 |
| 任务账本 | 任务板和任务卡片 | 真实任务账本 |
| 智能体身份 | 计划智能体、工作智能体、审查智能体状态摘要 | 完整智能体档案 |
| 持久消息与即时唤醒 | 底部事件流静态样例 | 真实任务消息 |
| 上下文恢复 | 任务详情页上下文、执行详情页上下文快照 | 真实上下文恢复 |
| 健康监控 | 右侧状态栏系统健康摘要、异常标签 | 健康页 |
| 验证闸门 | 执行详情页验证 Tab、审查包验证结果 | 真实验证策略 |
| 合并队列 | 不采用 | 后续合并队列 |
| 调度器 | 不采用 | 后续多任务并行 |
| 工作流模板 | 只做占位和接受结果后的提示 | 工作流库 |

## 不建议照搬的点

1. 西部小镇命名体系。
2. 复杂依赖。
3. 一开始就做大规模多智能体并发。
4. 一开始就做合并队列。
5. 一开始就做联邦协作。
6. 一开始就做真实安全权限系统。
7. 一开始就做完整命令行生态。

## 当前结论

Gas Town 只作为思想参考，不作为实现蓝本。

V0.1.1 不复制 Gas Town 的工程结构、命名体系或复杂调度能力。

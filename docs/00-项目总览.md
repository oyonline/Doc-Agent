# 项目总览

## 项目名称

智能体工作台。

## 一句话定位

智能体工作台是一个以任务板为中心、以智能体状态为侧栏、以执行证据为详情、以审查包为决策入口的智能体工作台。

本项目的核心不是和智能体聊天，而是管理智能体交付。

## 当前阶段

V0.1 可点击交互原型。

V0.1 只服务于可点击、可走查的交互原型，不服务于真实后端、真实智能体调用、数据库、部署或权限实现。

## 当前主线

项目记忆包 → 需求说明 V0.1.1 → 六页低保真原型 → 主流程走查 → 高保真视觉 → 前端静态演示。

## 当前最重要的产出

当前最重要的产出是 `docs/02-需求说明-V0.1.1.md`，它用于指导六页低保真可点击原型。

配套产出包括：

- `docs/09-低保真原型任务书.md`
- `docs/10-样例数据.md`
- `docs/11-页面跳转矩阵.md`
- `docs/12-文案统一表.md`

## 当前范围

V0.1 包含六个核心页面：首页、计划确认页、任务板、任务详情页、执行详情页、审查包页。

V0.1 还包含右侧状态栏、底部事件流、默认严格安全模式，以及工作流、智能体、设置三个占位页。

## 当前不做什么

- 不做真实智能体调用。
- 不做真实后端。
- 不做数据库。
- 不做权限系统。
- 不做登录。
- 不做计费。
- 不做多用户。
- 不做移动端。
- 不做大规模多智能体并发。
- 不做合并队列。
- 不做真实工作流库。

## 长期方向

长期方向是让用户可以稳定管理智能体交付：从目标输入、计划确认、任务执行、证据查看、验证结果、审查包决策，到成功经验沉淀为工作流。

未来可以逐步扩展工作流库、多任务并行、智能体档案、健康页、安全设置、任务消息、合并队列和长期经验沉淀。

## 如何使用项目记忆包

新会话开始时，先阅读 `docs/99-新会话启动提示词.md` 中列出的文件。推进过程中，以 `docs/02-需求说明-V0.1.1.md` 作为主需求来源，以 `docs/06-下一步行动.md` 作为下一步入口。

每次完成重要工作后，应同步更新任务清单、决策日志、对话摘要和下一步行动。

# V0.2 · 单一事实源

## 文档目的

V0.2 文档骨架由 9 份文档组成,后续代码 Agent 或新协作者在某些概念上可能产生"该信哪份"的困惑。本文档明确每个概念的事实源文档与具体章节,作为冲突时的唯一仲裁依据。

本文档不重新定义任何概念,只做映射与导航。

## 事实源原则

1. 每个概念有且只有一个事实源文档。
2. 事实源文档之外的文档可以引用,但不能重新定义。
3. 当发现两份文档对同一概念有不一致定义时,以事实源为准,另一份必须修正。
4. 决策日志(`docs/03-决策日志.md`)记录"为什么这么定",事实源文档记录"是什么"。
5. 事实源映射如有调整,必须先更新本文档,再修改其他引用文档。

## 容易混淆的概念辨析

### "决策日志"有两份,各有归属

- **`docs/03-决策日志.md`**:**项目级**决策日志,记录项目演进过程的决策。例如:为什么从 V0.1.1 走到 V0.2、为什么演示已驳回、为什么把样例数据策略统一为 7 任务版本。条目按时间序列累计,跨版本沿用。
- **`docs/06-V0.2-样例数据与事件文案.md` §决策日志**:**需求级**决策日志,记录智能体执行需求-001 时产生的关键决策。例如:为什么标签长度限制 20 字符、为什么用游标分页、为什么 DELETE 默认硬删除。条目仅在需求-001 样例的"任务详情页 → 决策与上下文 Tab(V0.2.2 启用)"中展示,不与项目级决策日志混合。

判断方法:如果决策跨越多个需求或属于"项目治理",归项目级;如果决策只服务于"需求-001 这次执行",归需求级。

### "异常标签" vs "异常路径"

- **异常标签**:任务/需求/运行的异常状态外化,叠加在主状态上。共 3 类(需求 3 个 / 任务 6 个 / 运行无独立异常标签)。事实源在 `docs/03`(各状态机异常标签子节)。
- **异常路径**:异常发生后的状态变化时序剧本(谁是触发者、状态怎么变、智能体怎么动、如何恢复)。共 6 条。事实源在 `docs/04 §异常路径`(逻辑)+ `docs/06 §11.4`(具体数据)。

判断方法:讨论"出问题后任务长什么样"看 docs/03 异常标签;讨论"出问题后系统怎么演进"看 docs/04 异常路径。

### "单任务审查包" vs "需求级审查包"

- **单任务审查包**:每个任务完成后生成,V0.1.1 资产沿用,展示任务回顾 / 总结 / 验收结果 / 文件清单 / diff / 验证 / 风险 / 建议。事实源在 `docs/06 §7.X`(各任务样例)+ `docs/03 跳转矩阵`(行为)。
- **需求级审查包**:全部任务完成后生成,V0.2 新增,展示需求回顾 / PRD 对比 / 任务汇总 / 跨任务一致性 / 风险 / 审查建议 / 决策选项。事实源在 `docs/06 §需求级审查包` + `docs/03 跳转矩阵 - 需求级审查包页`。

判断方法:粒度是任务还是需求。任务级是技术 review,需求级是产品验收。

### "上下文 Tab" vs "决策与上下文 Tab"

- **上下文 Tab**(V0.1.1 沿用,V0.2.0 / V0.2.1 阶段使用):任务详情页 4 Tab 之一(概览 / 上下文 / 验收标准 / 历史运行),只展示初始上下文(角色、任务、验收标准、项目规则、工作区、需要运行的验证、上一次反馈、安全模式)。
- **决策与上下文 Tab**(V0.2.2 启用):任务详情页 4 Tab 之一(同结构,Tab 名变更),内容扩展为三块——初始上下文 / 决策日志 / 会话分段记录。

判断方法:看当前实现批次。V0.2.0 / V0.2.1 阶段是"上下文",V0.2.2 阶段升级为"决策与上下文"。事实源:`docs/05 §UI 呈现规范`(V0.2.2 结构)+ `docs/03 §V0.2.0/V0.2.1/V0.2.2 批次差异说明`(批次差异)。

### "调度规则"(docs/02)vs "调度核心原则"(docs/04)

- **调度规则**(`docs/02 §4.4 项目管理智能体 → 调度规则`):5 条机械规则,定义项目管理智能体在调度行为中的"硬规则"(实线前置依赖、可调度任务同时入队、智能体可同时处理多任务、人工介入暂停、审查包不阻塞)。
- **调度核心原则**(`docs/04 §调度核心原则`):5 条核心原则,在 docs/02 5 条规则之上追加,定义 V0.2 调度的"产品意图"(依赖驱动不靠时间表、必须真并行、自动放行是默认、审查包不阻塞、异常路径不省略)。

判断方法:讨论"调度怎么发生"看 docs/02;讨论"调度为什么这么设计"看 docs/04。两者并列,不重复。

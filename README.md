# 智能体工作台

智能体工作台是一个以任务板为中心、以智能体状态为侧栏、以执行证据为详情、以审查包为决策入口的智能体工作台。

## 项目大目标

智能体工作台的长期目标，是把智能体工作从“聊天窗口里的临时执行”，变成“可管理、可观察、可验证、可审查、可复用的交付流程”。

它要解决的问题不是“如何让智能体回答得更好”，而是“如何让智能体可靠地完成任务，并让人类清楚判断结果是否可以接受”。

因此，本项目的核心不是和智能体聊天，而是管理智能体交付。

当前 V0.1.1 阶段只验证产品形态：通过六页可点击原型，走通“目标 → 计划 → 任务 → 运行 → 验证 → 审查包 → 人工决策”的最小闭环。

长期方向是成为面向智能体工作的交付控制台，而不是单纯任务看板。

## 当前阶段

V0.1.1 可点击、可走查的低保真交互原型。

## 当前主线

项目记忆包 → 需求说明 V0.1.1 → 六页低保真原型 → 主流程走查 → 高保真视觉 → 前端静态演示。

## 如何继续推进

1. 先阅读 `docs/00-项目总览.md`，确认项目定位和阶段边界。
2. 再阅读 `docs/02-需求说明-V0.1.1.md`，以它作为当前主需求文档。
3. 使用 `docs/09-低保真原型任务书.md` 开始绘制六页低保真可点击原型。
4. 使用 `docs/10-样例数据.md` 填充页面内容。
5. 使用 `docs/11-页面跳转矩阵.md` 和 `docs/12-文案统一表.md` 连接交互并统一按钮文案。

## 当前不做什么

当前 V0.1.1 不做真实后端、真实智能体调用、数据库、权限系统、计费、多用户、移动端、大规模多智能体调度、合并队列和真实工作流库。

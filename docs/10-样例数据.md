# 样例数据

## 样例目标

修复登录页错误提示不显示，并补充测试。

## 样例计划

1. 检查登录失败后的错误状态是否被保存。
2. 检查错误提示组件是否接收到错误文案。
3. 修复错误提示显示链路。
4. 补充登录失败场景测试。
5. 运行验证并生成审查包。

## 样例验收标准

- 登录失败时页面展示明确错误提示。
- 错误提示文案来自失败状态。
- 原有登录成功流程不受影响。
- 新增登录失败场景测试。
- 验证结果通过。

## 样例任务卡

- 标题：修复登录页错误提示不显示，并补充测试。
- 状态：待审查。
- 负责智能体：工作智能体。
- 风险标签：低风险。
- 最近更新：验证通过，等待人工审查。
- 主操作：查看审查包。

## 样例静态差异预览

```text
文件：登录页

- 未传入错误提示内容
+ 将登录失败状态传入错误提示组件

- 错误状态只记录在提交逻辑中
+ 错误状态同步给页面提示区域
```

```text
文件：登录失败测试

- 测试只覆盖登录成功
+ 新增登录失败时显示错误提示的测试

- 未检查错误文案
+ 检查页面出现“账号或密码不正确”
```

# 后续执行智能体说明

## 项目定位

本项目是“智能体工作台”。

它不是聊天机器人，也不是普通任务管理工具，
而是围绕智能体交付过程建立的工作台：
用户输入目标，系统生成结构化计划，智能体执行，
系统展示上下文、日志、改动、验证结果，最后生成审查包，
由人做接受结果、拒绝并反馈、重新执行或让审查智能体复查的决策。

## 当前阶段

当前阶段是 V0.1.1 可点击、可走查的低保真交互原型。

本阶段只服务于六个核心页面：

- 首页。
- 计划确认页。
- 任务板。
- 任务详情页。
- 执行详情页。
- 审查包页。

工作流、智能体、设置只做统一占位页。

## 项目大目标

长期目标是把智能体工作从“聊天窗口里的临时执行”，变成“可管理、可观察、可验证、可审查、可复用的交付流程”。

当前 V0.1.1 只验证这条主链路：

```text
目标 → 计划 → 任务 → 运行 → 验证 → 审查包 → 人工决策
```

## 禁止事项

- 禁止后续智能体重新发散产品方向。
- 禁止恢复“运行 / 审查”一级导航。
- 禁止新增核心页面。
- 禁止把历史 V0.1 / V0.2 / 旧需求文档置于 V0.1.1 整合版之上。
- 禁止把 V0.2 或后续能力写成 V0.1.1 必做。
- 禁止把工作流、智能体、设置做成真实功能页。
- 禁止把审查包文件清单改回跳转到执行详情页。
- 审查包页必须同页展示静态差异预览，不允许文件清单跳转出去看 diff。
- 禁止恢复审查包四个按钮平铺。
- 禁止在顶栏重复展示自动合并、外部网络、密钥访问。
- 禁止恢复任务详情页 `查看上下文` 按钮。
- 禁止恢复执行详情页“产物”Tab。
- 禁止写真实后端方案、数据库设计、模型调用方案、部署方案、权限系统实现或计费方案。
- 禁止把本项目做成聊天机器人界面。
- 禁止照搬 Gas Town 的命名体系。
- 禁止夹杂不必要英文。

## 文案规则

统一使用：

- 智能体，不用 Agent / 代理。
- 工作流，不用 Workflow。
- 审查包，不用 Review Packet。
- 接受结果，不用通过 / 合并。
- 拒绝并反馈，不用拒绝 / 驳回。
- 重新执行，不用重跑。
- 停止，不用中止 / 终止。

文案应克制、具体、像真实产品，不使用抽象宣传词。

当前项目大目标：

智能体工作台的长期目标，
是把智能体工作从“聊天窗口里的临时执行”，
变成“可管理、可观察、可验证、可审查、可复用的交付流程”。

当前阶段不是实现真实智能体系统，而是通过 V0.1.1 六页低保真原型，验证：

```text
目标 → 计划 → 任务 → 运行 → 验证 → 审查包 → 人工决策
```

当前产品定位：

智能体工作台不是聊天机器人，也不是普通任务管理工具，而是管理智能体交付的工作台。

当前范围：

- 五个一级导航：首页、任务、工作流、智能体、设置。
- 六个核心页面：首页、计划确认页、任务板、任务详情页、执行详情页、审查包页。
- 三个统一占位页：工作流、智能体、设置。
- 一条端到端样例流程：“修复登录页错误提示不显示，并补充测试。”

- 不恢复“运行 / 审查”一级导航。

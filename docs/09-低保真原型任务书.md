# 低保真原型任务书

## 原型目标

走通“修复登录页错误提示不显示，并补充测试”的完整路径。

低保真原型需要让用户从首页输入目标开始，经过计划确认、任务详情、执行详情和审查包，最终接受结果并回到任务板的已完成状态。

## 需要画的页面

1. 首页。
2. 计划确认页。
3. 任务板。
4. 任务详情页。
5. 执行详情页。
6. 审查包页。
7. 工作流占位页。
8. 智能体占位页。
9. 设置占位页。

核心可点击路径只要求前六页。后三页作为导航占位。

## 跳转要求

页面连线以 `docs/11-页面跳转矩阵.md` 为准。

重点连线：

- 首页点击“生成任务计划”进入计划确认页。
- 计划确认页点击“确认并执行”进入任务详情页。
- 任务详情页点击“查看运行详情”进入执行详情页。
- 执行详情页点击“查看审查包”进入审查包页。
- 审查包页点击修改文件清单时，只更新同页差异预览，不跳转。
- 审查包页点击“接受结果”后返回任务板，任务进入已完成。

## 低保真验收标准

- 六页可访问。
- 主路径无断点。
- 审查包可完成决策。
- 右侧状态栏常驻。
- 底部事件流常驻。
- 审查包支持同页差异预览。
- 占位页文案统一。
- 操作文案与 `docs/12-文案统一表.md` 一致。

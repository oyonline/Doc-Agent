# 项目总览 · V0.1.1 整合版

## 一句话定位

智能体工作台是一个管理智能体交付的工作台，不是聊天机器人，也不是普通任务管理工具。

它的核心主轴是任务，而不是对话。用户输入目标，系统生成计划，用户确认计划后智能体执行，系统展示上下文、日志、改动和验证结果，最后生成审查包，由用户接受结果、拒绝并反馈、重新执行或让审查智能体复查。

## 当前版本目标

当前版本是 V0.1.1 整合版，目标是产出一套可点击、可走查的低保真交互原型。

这版只验证产品心智和主链路是否成立：

```text
目标 → 计划 → 任务 → 运行 → 验证 → 审查包 → 人工决策
```

当前阶段不做真实智能体系统，也不做真实技术实现。

## 五个一级导航

左侧导航只保留：

- 首页。
- 任务。
- 工作流。
- 智能体。
- 设置。

不保留：

- 运行。
- 审查。

“运行”归入任务详情页的“历史运行”和执行详情页。

“审查”归入任务板的“待审查”列和审查包页。

## 六个核心页面

V0.1.1 只有六个核心页面：

1. 首页。
2. 计划确认页。
3. 任务板。
4. 任务详情页。
5. 执行详情页。
6. 审查包页。

不要新增运行列表页、审查队列页、健康页、设置详情页、工作流库页等核心页面。

## 三个占位页

工作流、智能体、设置三个页面在 V0.1.1 只做统一占位。

统一结构：

- 页面标题。
- 说明文案。
- “V0.1.1 暂未开放”提示。
- `返回首页` 按钮。

不要展开真实功能，不要设计二级导航。

## 样例流程

贯穿样例目标：

> 修复登录页错误提示不显示，并补充测试。

端到端流程：

1. 首页输入目标。
2. 点击 `生成任务计划`。
3. 计划确认页展示 6 步计划与 3 条验收标准。
4. 点击 `确认并执行`。
5. 进入任务详情页，任务状态为执行中。
6. 查看上下文。
7. 点击 `查看运行详情`。
8. 进入执行详情页。
9. 查看时间线、上下文快照、日志、改动、验证。
10. 验证全部通过后进入审查包。
11. 在审查包页查看总结、验收标准、改动差异、验证结果、风险提示、审查建议。
12. 点击 `接受结果`。
13. 弹出“是否保存为工作流？”提示。
14. 选择 `暂不保存`。
15. 任务进入已完成列。
16. 事件流出现“任务-001 已接受”。

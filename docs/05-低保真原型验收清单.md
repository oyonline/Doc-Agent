# 低保真原型验收清单

## 使用说明

本清单用于验收 V0.1.1 可点击、可走查的低保真交互原型。

验收时只检查 V0.1.1 范围内内容，不检查真实后端、数据库、模型调用、登录、权限、计费、多用户、移动端、真实 Git / GitHub 集成、真实工作流库、真实智能体档案或真实设置页。

## 导航与范围

- [ ] 左侧导航只有：首页、任务、工作流、智能体、设置。
- [ ] 不出现“运行”一级导航。
- [ ] 不出现“审查”一级导航。
- [ ] 不出现独立运行列表页。
- [ ] 不出现独立审查队列页。
- [ ] 不出现健康页、设置详情页、工作流库页作为 V0.1.1 核心页面。
- [ ] 工作流、智能体、设置只做占位页。

## 六个核心页面

- [ ] 首页可访问。
- [ ] 计划确认页可访问。
- [ ] 任务板可访问。
- [ ] 任务详情页可访问。
- [ ] 执行详情页可访问。
- [ ] 审查包页可访问。
- [ ] 六个核心页面之间的跳转符合 `docs/03-状态机与跳转矩阵.md`。

## 样例流程

- [ ] 首页可以输入“修复登录页错误提示不显示，并补充测试。”
- [ ] 点击 `生成任务计划` 后进入计划确认页。
- [ ] 计划确认页展示 6 步计划。
- [ ] 计划确认页展示 3 条验收标准。
- [ ] 点击 `确认并执行` 后进入任务详情页。
- [ ] 任务详情页状态为执行中。
- [ ] 任务详情页可以查看上下文 Tab。
- [ ] 点击 `查看运行详情` 后进入执行详情页。
- [ ] 执行详情页可以查看时间线、上下文快照、日志、改动、验证。
- [ ] 验证全部通过后可以进入审查包页。
- [ ] 审查包页可以查看总结、验收标准、改动差异、验证结果、风险提示、审查建议。
- [ ] 点击 `接受结果` 后弹出“是否保存为工作流？”提示。
- [ ] 选择 `暂不保存` 后回到任务板。
- [ ] 任务进入已完成列。
- [ ] 事件流出现“任务-001 已接受”。
- [ ] 选择 `保存为工作流` 后进入工作流占位页。
- [ ] 工作流占位页展示“前端问题修复”静态示意卡。
- [ ] 事件流出现“已保存工作流：前端问题修复”。

## 审查包页

- [ ] 审查包页文件清单默认选中第一个文件。
- [ ] 审查包页文件清单点击后同页展示差异。
- [ ] 点击文件清单不跳转到执行详情页。
- [ ] 至少展示一个静态 diff 样例。
- [ ] 审查包底部固定决策区存在。
- [ ] 底部左侧为 `拒绝并反馈`。
- [ ] 底部右侧为 `接受结果`。
- [ ] `更多操作` 中包含 `重新执行`。
- [ ] `更多操作` 中包含 `让审查智能体复查`。
- [ ] `拒绝并反馈` 必须填写反馈。
- [ ] `重新执行` 不要求填写反馈。
- [ ] `让审查智能体复查` 后主状态保持待审查。

## 全局组件

- [ ] 右侧状态栏所有页面常驻。
- [ ] 右侧状态栏不可隐藏。
- [ ] 右侧状态栏展示“安全模式：严格”。
- [ ] 右侧状态栏展示“自动合并：关闭”。
- [ ] 右侧状态栏展示“外部网络：受限”。
- [ ] 右侧状态栏展示“密钥访问：禁止”。
- [ ] 顶栏只展示“严格模式”小标签。
- [ ] 顶栏不展示自动合并、外部网络、密钥访问细项。
- [ ] 底部事件流所有页面常驻。
- [ ] 底部事件流可折叠。

## 状态与标签

- [ ] 任务状态只使用五个主状态：待处理、执行中、验证中、待审查、已完成。
- [ ] 异常状态只作为标签展示。
- [ ] 异常标签不作为任务板列。
- [ ] 可展示的异常标签包括：已停止、执行失败、验证未通过、已拒绝、卡住、需要人工处理。
- [ ] 运行状态只用于历史运行和执行详情页。
- [ ] 验证状态只使用：未开始、进行中、已通过、未通过。
- [ ] 智能体状态只使用：空闲 / 工作中。

## 任务详情页

- [ ] 任务详情页顶部按钮只保留 `查看运行详情`、`停止`、`重试`、`查看审查包`。
- [ ] 任务详情页不出现 `查看上下文` 按钮。
- [ ] 任务详情页 Tab 为：概览、上下文、验收标准、历史运行。
- [ ] 历史运行列表可进入执行详情页。

## 执行详情页

- [ ] 执行详情页 Tab 只保留：时间线、上下文快照、日志、改动、验证。
- [ ] 执行详情页不出现“产物”Tab。
- [ ] 改动 Tab 展示修改文件汇总。
- [ ] 改动 Tab 展示修改文件列表。
- [ ] 改动 Tab 展示至少一个静态差异样例。
- [ ] 改动 Tab 展示改动摘要。
- [ ] 不出现 `丢弃改动` 按钮。
- [ ] 验证未通过时 `查看审查包` 置灰。
- [ ] 验证未通过时提示：“验证未通过时无法查看审查包，请先重试或人工查看日志。”

## 占位页

- [ ] 工作流占位页默认包含标题、说明文案、“V0.1.1 暂未开放”提示、`返回首页`。
- [ ] 接受结果并选择保存后，工作流占位页可额外展示“已保存工作流”静态示意卡。
- [ ] 智能体占位页只有标题、说明文案、“V0.1.1 暂未开放”提示、`返回首页`。
- [ ] 设置占位页只有标题、说明文案、“V0.1.1 暂未开放”提示、`返回首页`。
- [ ] 占位页不设计二级导航。
- [ ] 占位页不展开真实功能。

## 文案

- [ ] 使用“智能体”，不使用 Agent / 代理。
- [ ] 使用“工作流”，不使用 Workflow。
- [ ] 使用“审查包”，不使用 Review Packet。
- [ ] 使用 `接受结果`，不使用通过 / 合并。
- [ ] 使用 `拒绝并反馈`，不使用拒绝 / 驳回。
- [ ] 使用 `重新执行`，不使用重跑。
- [ ] 使用 `停止`，不使用中止 / 终止。
- [ ] 文案克制、具体，不使用抽象宣传词。

## 完成判定

全部必检项通过后，V0.1.1 低保真原型可以进入主流程走查。

走查完成后，再决定是否进入高保真视觉或前端静态演示。

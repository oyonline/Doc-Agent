# V0.2 · 验收清单

## 文档目的

本文档是 V0.2 实现的验收依据,从 `docs/00`、`docs/02`、`docs/03`、`docs/04`、`docs/05`、`docs/06` 机械汇总验收点。后续代码 Agent 完成实现后,以本清单为准检查"做完没做完"。

本文档不重新定义任何概念,只把已锁定的范围、状态、调度、数据、文案转换为可勾选的检查项。每条验收项标明来源文档与章节。

## 验收原则

1. 按 V0.2.0 / V0.2.1 / V0.2.2 三批分阶段验收。
2. 每条验收项指明来源文档和章节;争议时回到来源文档以源为准。
3. 验收项分 6 类:页面级 / 状态级 / 调度级 / Context 级 / 样例数据级 / 事件流级。智能体级单独列出作为状态级的补充。
4. 每批的验收清单是该批"实现完成可进入下一批"的硬门槛,任何一项不通过都不允许进入下一批。
5. 跨批次能力(V0.2.0 不能含 V0.2.2 的 Context Tab)即使部分实现也算违规,验收时直接打回。

## 严禁事项

- 严禁让某个验收项跨批次混搭(V0.2.0 验收项不能要求 V0.2.2 的 Context Tab)。
- 严禁让验收项的"来源"模糊或缺失(每条验收项必须能追溯到具体文档和章节)。
- 严禁让验收清单成为"已实现的功能描述",它必须是"待勾选的检查项"(用 [ ] 而非 ✓)。
- 严禁在验收清单里重新定义概念,如果与来源文档不一致,以来源文档为准。
- 严禁跳过任何验收项进入下一批(部分实现 + 跳过 = 违规)。

# 智能体工作台 · 需求说明 V0.1.1

## 1. 文档目的

这份文档用于产出一个可点击、可走查的 V0.1 交互原型。它面向产品设计、低保真线框、页面连线和主流程走查，帮助设计师或界面生成工具明确页面结构、状态、样例内容、跳转关系和统一文案。

这份文档不用于技术实现，不包含真实后端、数据库、模型调用、部署、权限、计费、多用户协作或移动端方案。文档中的所有数据均为原型样例数据，用于表达产品心智和交互路径。

## 2. 产品定位

智能体工作台不是聊天机器人，而是智能体交付管理工作台。用户不是来和智能体闲聊，而是来管理从目标到交付的完整过程。

一句话定位：

“一个以任务板为中心、以智能体状态为侧栏、以执行证据为详情、以审查包为决策入口的智能体工作台。”

它解决的问题是：当智能体参与完成任务时，用户需要清楚知道目标是什么、计划是什么、正在执行什么、使用了哪些上下文、产生了哪些改动、验证是否通过、风险是什么，以及最终是否可以接受结果。

### 项目大目标与 V0.1.1 的关系

智能体工作台的长期目标，是把智能体工作从“聊天窗口里的临时执行”，变成“可管理、可观察、可验证、可审查、可复用的交付流程”。

当前 V0.1.1 不试图实现完整系统，只验证这条交付链路的产品形态是否成立：

目标 → 计划 → 任务 → 运行 → 验证 → 审查包 → 人工决策。

因此，V0.1.1 的重点不是增加功能，而是让用户清楚看到：

- 智能体要做什么。
- 智能体看到了什么上下文。
- 智能体执行到了哪一步。
- 智能体改了什么。
- 验证有没有通过。
- 风险在哪里。
- 用户是否应该接受结果。

这份文档仍然是 V0.1.1 低保真交互原型需求，不把长期目标里的后续能力写成 V0.1.1 必做项。

## 3. 目标用户与典型场景

### 目标用户

- 个人开发者：希望把明确目标交给智能体执行，并需要可审查的结果。
- 小团队成员：希望让智能体参与日常交付，但需要团队能看懂过程和结果。
- 对智能体产出有审查和合入决策权的人：需要基于证据做接受、拒绝、复查或重新执行。

### 典型场景

- 输入目标并让系统拆解：用户输入“修复登录页错误提示不显示，并补充测试”，系统生成结构化计划和验收标准。
- 查看任务执行状态：用户在任务板看到任务从待处理进入执行中、验证中、待审查和已完成。
- 查看上下文、日志、改动和验证结果：用户进入任务详情页和执行详情页，确认智能体做了什么、依据什么做、验证是否通过。
- 基于审查包做最终决策：用户在审查包页查看同页差异预览、验证结果和风险提示，然后接受结果。
- 拒绝后反馈进入下一次运行上下文：如果用户拒绝，必须填写反馈，反馈会出现在下一次运行的上下文快照里。

## 4. V0.1 范围

### 包含

- 整体外壳。
- 六个核心页面。
- 一条端到端样例流程。
- 右侧状态栏。
- 底部事件流。
- 默认严格安全模式。
- 工作流、智能体、设置三个占位页。

六个核心页面为：

1. 首页。
2. 计划确认页。
3. 任务板。
4. 任务详情页。
5. 执行详情页。

### 不包含

- 真实后端。
- 数据库。
- 权限系统。
- 登录。
- 计费。
- 多用户。
- 移动端。
- 大规模多智能体并发。
- 合并队列。
- 真实工作流库。

## 5. 设计原则

V0.1 采用以下设计原则：

- 任务优先于对话：主界面围绕任务板和任务详情，不使用聊天流作为中心。
- 状态必须外化：任务状态、运行状态、安全状态和系统健康要始终可见。
- 输入也要可审查：原始目标、建议计划、验收标准和人工反馈都要成为可查看上下文。
- 审查包是最终决策入口：接受、拒绝、复查和重新执行都从审查包发生。
- 安全状态常驻：严格模式、自动合并关闭、外部网络受限、密钥访问禁止要持续展示。
- 人类负责最终接受与拒绝：智能体可以建议，但不能替代人的最终决策。
- V0.1 保持克制：只做可点击原型，不扩展真实系统能力。
- 中文优先，非必要不夹杂英文：按钮、状态、说明文案统一中文表达。
- 不照搬 Gas Town 命名体系：只借鉴思想，不使用其角色名作为本项目术语。
- 从目标到交付必须形成闭环：用户要能从输入目标一路走到接受结果。

## 6. 整体信息架构

V0.1 左侧导航只保留：

- 首页。
- 任务。
- 工作流。
- 智能体。
- 设置。

“运行”不作为一级导航。“运行”放入任务详情页的历史运行和执行详情页。

“审查”不作为一级导航。“审查”放入任务板的“待审查”列和审查包页。

### 整体布局

顶部：项目名、严格模式小标签、创建目标按钮。

左侧：五个导航入口：首页、任务、工作流、智能体、设置。

中间：主工作区，展示当前页面内容。

右侧：智能体状态、系统健康、安全模式。

底部：事件流，可折叠。

## 7. 全局组件

### 顶栏

顶栏展示项目名“智能体工作台”、严格模式小标签和“创建目标”按钮。顶栏不展示详细安全信息，详细安全信息放入右侧状态栏。

### 左侧导航

左侧导航包含：首页、任务、工作流、智能体、设置。当前页面需要有清晰选中态。

### 右侧状态栏

右侧状态栏必须始终展示：

- 安全模式：严格。
- 自动合并：关闭。
- 外部网络：受限。
- 密钥访问：禁止。
- 提示文案：“智能体可以执行任务，接受结果与合入需要人确认。”

右侧状态栏还可以展示计划智能体、工作智能体、审查智能体的占位状态，以及系统健康概览。

### 底部事件流

底部事件流展示最近关键事件，默认展开，支持折叠。事件内容使用静态样例，不需要实时更新。

### 任务卡片

任务卡片展示任务标题、当前状态、负责智能体、风险标签、最近更新时间和主要操作入口。

### 状态标签

主状态使用：待处理、执行中、验证中、待审查、已完成。

异常标签使用：已停止、执行失败、验证未通过、已拒绝、卡住、需要人工处理。

### 风险提示

风险提示用于提示执行边界、验证风险、需要人工确认的事项。风险提示不替代最终决策。

### 二次确认弹窗

危险操作需要二次确认，包括停止、接受结果、重新执行。拒绝并反馈需要填写反馈，不用额外二次确认。

### 占位页

工作流、智能体、设置三个页面为占位页，使用统一结构：页面标题、说明文案、“V0.1 暂未开放”提示、返回首页按钮。

## 8. 核心页面说明

### 8.1 首页

页面目的：让用户输入目标，看到当前任务概览和最近事件，并进入计划确认流程。

进入方式：左侧导航点击“首页”；占位页点击“返回首页”；顶栏点击项目名也可回到首页。

页面结构：上方为目标输入区，中部为当前任务概览，下方为最近事件摘要。右侧状态栏和底部事件流常驻。

主要组件：

- 目标输入框。
- “生成任务计划”按钮。
- 当前任务概览区。
- 最近事件列表。
- 严格模式提示。

关键交互：

- 用户输入目标后点击“生成任务计划”，进入计划确认页。
- 用户点击当前任务概览中的任务卡，进入任务详情页。
- 用户点击左侧“工作流”“智能体”“设置”，进入对应占位页。

页面状态：

- 正常态：目标输入区可用，任务概览展示样例任务，最近事件展示三条事件。
- 空状态：没有任务时展示“还没有任务，先输入一个目标开始”。
- 加载态：点击“生成任务计划”后展示“正在生成任务计划”。
- 异常态：计划生成失败时展示“任务计划生成失败，请检查目标描述后重试”。
- 需要人工处理的状态：目标过于模糊时提示“请补充预期结果或验收标准”。

跳转关系：

- 首页 → 计划确认页：点击“生成任务计划”。
- 首页 → 任务详情页：点击当前任务卡。
- 首页 → 占位页：点击工作流、智能体、设置。

样例文案：

- 输入框提示：“输入你希望智能体完成的目标。”
- 样例目标：“修复登录页错误提示不显示，并补充测试。”
- 按钮：“生成任务计划”。

### 8.2 计划确认页

页面目的：让用户审查系统建议计划和验收标准，确认后再开始执行。

进入方式：首页输入目标并点击“生成任务计划”。

页面结构：顶部展示原始目标回显，中部展示系统建议计划和验收标准，底部展示操作区。右侧状态栏和底部事件流常驻。

主要组件：

- 原始目标回显。
- 系统建议计划。
- 验收标准列表。
- 风险提示。
- “确认并执行”按钮。
- “保存草稿”按钮。

关键交互：

- 点击“确认并执行”，任务进入执行中，并跳转任务详情页。
- 点击“保存草稿”，任务保持待处理，并返回首页或任务板。
- 修改计划内容在 V0.1 中不做真实编辑，只展示静态计划。

页面状态：

- 正常态：完整展示目标、计划和验收标准。
- 空状态：未接收到目标时提示“缺少原始目标，请返回首页重新输入”。
- 加载态：展示“正在整理建议计划”。
- 异常态：展示“建议计划生成失败，请返回首页重试”。
- 需要人工处理的状态：验收标准不足时提示“建议补充可验证标准后再执行”。

跳转关系：

- 计划确认页 → 任务详情页：点击“确认并执行”。
- 计划确认页 → 首页：点击返回或保存草稿。

样例文案：

- 原始目标：“修复登录页错误提示不显示，并补充测试。”
- 系统建议计划标题：“建议计划”。
- 主按钮：“确认并执行”。

### 8.3 任务板

页面目的：展示任务在主状态之间的流转，让用户快速进入任务详情或审查包。

进入方式：左侧导航点击“任务”；审查包页点击“返回任务板”。

页面结构：中间为五列看板，列名依次为待处理、执行中、验证中、待审查、已完成。右侧状态栏和底部事件流常驻。

主要组件：

- 五列任务看板。
- 任务卡片。
- 状态标签。
- 风险标签。
- 顶部筛选占位。

关键交互：

- 点击任务卡进入任务详情页。
- 点击待审查任务卡上的“查看审查包”进入审查包页。
- 点击顶栏“创建目标”返回首页目标输入区。

页面状态：

- 正常态：五列展示样例任务流转状态。
- 空状态：某列没有任务时展示“暂无任务”。
- 加载态：展示“正在加载任务板”。
- 异常态：展示“任务板加载失败，请稍后重试”。
- 需要人工处理的状态：任务卡显示“需要人工处理”标签。

跳转关系：

- 任务板 → 任务详情页：点击任务卡。
- 任务板 → 审查包页：点击“查看审查包”。

样例文案：

- 列名：“待处理”“执行中”“验证中”“待审查”“已完成”。
- 卡片标题：“修复登录页错误提示不显示，并补充测试”。

### 8.4 任务详情页

页面目的：展示单个任务的目标、计划、上下文、验收标准和历史运行，并提供进入运行详情和审查包的入口。

进入方式：任务板点击任务卡；计划确认页点击“确认并执行”；审查包页点击返回任务详情。

页面结构：顶部为任务标题、状态和操作按钮；中部为标签页；右侧状态栏和底部事件流常驻。

标签为：

- 概览。
- 上下文。
- 验收标准。
- 历史运行。

任务详情页顶部按钮保留：

- 查看运行详情。
- 停止。
- 重试。
- 查看审查包。

不要放“查看上下文”按钮，因为上下文本身就是任务详情页的标签。

主要组件：

- 任务标题与状态。
- 操作按钮区。
- 标签页。
- 概览信息。
- 上下文内容。
- 验收标准结果。
- 历史运行列表。

关键交互：

- 点击“查看运行详情”进入执行详情页。
- 点击“停止”弹出二次确认，确认后任务显示已停止标签。
- 点击“重试”进入执行中状态。
- 点击“查看审查包”进入审查包页。
- 点击标签切换同页内容，不发生跨页跳转。

页面状态：

- 正常态：展示任务信息和当前运行摘要。
- 空状态：历史运行为空时展示“暂无历史运行”。
- 加载态：展示“正在加载任务详情”。
- 异常态：展示“任务详情加载失败”。
- 需要人工处理的状态：显示“需要人工处理”风险提示，并保留可进入审查包或重试的入口。

跳转关系：

- 任务详情页 → 执行详情页：点击“查看运行详情”。
- 任务详情页 → 审查包页：点击“查看审查包”。
- 任务详情页 → 任务板：通过左侧导航返回任务。

样例文案：

- 标题：“修复登录页错误提示不显示，并补充测试”。
- 标签：“概览”“上下文”“验收标准”“历史运行”。
- 按钮：“查看运行详情”“停止”“重试”“查看审查包”。

### 8.5 执行详情页

页面目的：展示一次运行中的时间线、上下文快照、日志、改动和验证结果，让用户理解智能体执行过程。

进入方式：任务详情页点击“查看运行详情”。

页面结构：顶部为运行标题和状态；中部为标签页；右侧状态栏和底部事件流常驻。

标签为：

- 时间线。
- 上下文快照。
- 日志。
- 改动。
- 验证。

不要放“产物”标签，V0.1 先砍掉。

主要组件：

- 运行状态摘要。
- 时间线列表。
- 上下文快照。
- 日志片段。
- 修改文件清单。
- 验证结果。
- “查看审查包”按钮。

关键交互：

- 点击标签切换执行证据。
- 点击“查看审查包”进入审查包页。
- 验证通过后任务进入待审查状态。

页面状态：

- 正常态：展示时间线、上下文、日志、改动和验证结果。
- 空状态：某个标签暂无内容时展示“暂无记录”。
- 加载态：展示“正在加载运行详情”。
- 异常态：展示“运行详情加载失败”。
- 需要人工处理的状态：验证未通过或日志显示阻塞时展示“需要人工处理”提示。

跳转关系：

- 执行详情页 → 审查包页：点击“查看审查包”。
- 执行详情页 → 任务详情页：返回上一级。

样例文案：

- 运行标题：“第 1 次运行”。
- 验证结果：“验证通过，建议进入审查包。”
- 按钮：“查看审查包”。

### 8.6 审查包页

页面目的：作为最终决策入口，让用户在同一页面查看任务回顾、智能体总结、验收标准、改动、验证和风险，并决定接受、拒绝、复查或重新执行。

进入方式：任务板点击待审查任务的“查看审查包”；任务详情页点击“查看审查包”；执行详情页点击“查看审查包”。

页面结构：顶部为审查包标题和任务状态；主体左侧为审查信息，右侧或下方为同页差异预览区域；底部为决策操作区。右侧状态栏和底部事件流常驻。

必须包含：

- 任务回顾。
- 智能体总结。
- 验收标准结果。
- 修改文件清单。
- 同页差异预览。
- 验证结果。
- 风险提示。
- 审查建议。
- 决策操作区。

审查包必须支持同页查看改动。修改文件清单点击后不要跳走。点击文件后，在同页右侧抽屉或下方区域展示静态差异预览。

审查包底部按钮层级为：

- 左侧：拒绝并反馈。
- 右侧：接受结果。
- 其他操作放入“更多操作”。

更多操作包含：

- 让审查智能体复查。
- 重新执行。

主要组件：

- 审查包摘要。
- 验收标准结果列表。
- 修改文件清单。
- 静态差异预览。
- 验证结果。
- 风险提示。
- 审查建议。
- 决策按钮。
- 反馈输入弹窗。
- 二次确认弹窗。

关键交互：

- 点击修改文件清单中的文件，更新同页差异预览。
- 点击“接受结果”，弹出二次确认；确认后任务进入已完成。
- 点击“拒绝并反馈”，打开反馈输入；必须填写反馈，提交后任务回到执行中。
- 点击“更多操作”中的“让审查智能体复查”，任务保持待审查并展示复查中提示。
- 点击“更多操作”中的“重新执行”，弹出二次确认；确认后任务回到执行中，不带新反馈。

页面状态：

- 正常态：完整展示审查信息和默认选中文件差异。
- 空状态：没有改动时展示“本次运行未产生可展示改动”。
- 加载态：展示“正在生成审查包”。
- 异常态：展示“审查包生成失败，请返回任务详情查看运行证据”。
- 需要人工处理的状态：风险提示或验收未完全通过时展示“需要人工判断是否接受”。

跳转关系：

- 审查包页 → 任务板：点击“返回任务板”或接受结果后返回。
- 审查包页 → 任务详情页：点击任务标题或返回任务详情。
- 审查包页 → 同页差异预览：点击修改文件清单，不离开当前页。

样例文案：

- 标题：“审查包：修复登录页错误提示不显示，并补充测试”。
- 审查建议：“建议接受。错误提示已传入组件，补充了失败场景测试，验证通过。”

- 左侧按钮：“拒绝并反馈”。
- 菜单：“更多操作”。

## 9. 端到端样例流程

唯一样例目标：

“修复登录页错误提示不显示，并补充测试。”

完整路径：

1. 用户进入首页，在目标输入区输入“修复登录页错误提示不显示，并补充测试。”
2. 用户点击“生成任务计划”，进入计划确认页。
3. 计划确认页回显原始目标，展示系统建议计划和验收标准。
4. 用户点击“确认并执行”，任务进入执行中，并跳转任务详情页。
5. 用户在任务详情页查看任务概览、上下文、验收标准和历史运行。
6. 用户点击“查看运行详情”，进入执行详情页。
7. 执行详情页展示时间线、上下文快照、日志、改动和验证。
8. 验证通过后，任务进入待审查状态。
9. 用户点击“查看审查包”，进入审查包页。
10. 用户点击修改文件清单，在同页查看静态差异预览。
11. 用户查看验证结果、风险提示和审查建议。
12. 用户点击“接受结果”，二次确认后任务进入已完成。
13. 页面返回任务板，任务出现在已完成列。

## 10. 状态机

任务主状态只保留五个：

- 待处理。
- 执行中。
- 验证中。
- 待审查。
- 已完成。

异常状态作为标签展示，不作为看板列：

- 已停止。
- 执行失败。
- 验证未通过。
- 已拒绝。
- 卡住。
- 需要人工处理。

### 状态流转

- 待处理 → 执行中。
- 执行中 → 验证中。
- 验证中 → 待审查。
- 待审查 → 已完成。
- 执行中 → 执行失败。
- 验证中 → 验证未通过。
- 待审查 → 拒绝并反馈 → 执行中。
- 待审查 → 重新执行 → 执行中。
- 任意可运行状态 → 停止 → 已停止标签。

“拒绝并反馈”必须填写反馈，反馈进入下一次运行上下文。

“重新执行”不带新反馈，按原计划重新跑一次。

# 智能体工作台 · 单一事实源 V0.1.1

> 本文是当前项目唯一事实源。历史 V0.1、V0.2、旧任务清单、旧跳转矩阵、旧文案表只作为背景资料；凡与本文冲突，均以本文为准。

## 文档目的

这份文档用于指导 V0.1.1 可点击、可走查的低保真交互原型。

它用于：

- 对齐产品定位、范围和页面结构。
- 指导设计师绘制六个核心页面和三个占位页。
- 指导前端或原型搭建者连接静态跳转。
- 统一状态、样例数据、按钮文案和验收标准。

它不用于：

- 真实后端、数据库、模型调用或部署方案。
- 登录、权限、计费、多用户设计。
- 真实 Git / GitHub 集成。
- 工作流库、智能体档案、真实设置页。
- V0.2 或后续版本功能展开。

## 产品定位

智能体工作台是一个管理智能体交付的工作台。

它不是聊天机器人，也不是普通任务管理工具。

核心主轴是任务，不是对话。

主流程是：

```text
用户输入目标
→ 系统生成计划
→ 用户确认计划
→ 智能体执行
→ 系统展示上下文、日志、改动、验证
→ 生成审查包
→ 用户接受、拒绝、重新执行或复查
```

它要解决的问题是：当智能体参与完成任务时，用户能清楚知道目标是什么、计划是什么、正在执行什么、使用了哪些上下文、产生了哪些改动、验证是否通过、风险是什么，以及最终是否可以接受结果。

## 目标用户

- 个人开发者：希望把明确目标交给智能体执行，并需要可审查的结果。
- 小团队成员：希望让智能体参与日常交付，但团队仍能看懂过程和结果。
- 有审查和接受决策权的人：需要基于证据接受结果、拒绝并反馈、重新执行或让审查智能体复查。

## 全局组件

### 顶栏

顶栏只展示：

- 产品名 / 项目名。
- 全局搜索占位。
- 不可点击的“严格模式”小标签。
- `创建目标`。
- 用户头像占位。

顶栏不要重复展示：

- 自动合并。
- 外部网络。
- 密钥访问。

这些安全信息只在右侧状态栏完整展示。

### 右侧状态栏

右侧状态栏在所有页面常驻，不可隐藏。

固定展示：

- 安全模式：严格。
- 自动合并：关闭。
- 外部网络：受限。
- 密钥访问：禁止。

同时展示智能体状态：

- 计划智能体：空闲 / 工作中。
- 工作智能体：空闲 / 工作中。
- 审查智能体：空闲 / 工作中。

说明文案：

> 智能体可以执行任务，接受结果与合入需要你确认。

### 底部事件流

底部事件流展示最近关键事件，默认展开，可折叠为单行最新事件。事件使用静态样例，不要求实时能力。

### 任务卡片

任务卡片字段：

- 任务名称。
- 主任务状态。
- 异常标签。
- 负责智能体。
- 已运行时长。
- 验证结果。
- 风险等级。

默认展示任务名称、主任务状态、负责智能体、已运行时长；悬浮或展开时展示验证结果、风险等级和异常标签。

## 六个核心页面

### 1. 首页

目的：让用户输入目标，并看到当前任务概览。

必须包含：

- 目标输入区。
- `生成任务计划` 按钮。
- 当前任务概览。
- 最近事件。
- 右侧状态栏。
- 底部事件流。

关键交互：

- 输入目标后点击 `生成任务计划`，进入计划确认页。
- 顶栏点击 `创建目标`，回到首页并聚焦目标输入框。
- 点击当前任务卡片，进入任务详情页。

V0.1.1 可以保留 `只创建任务` 作为文本按钮，但它只创建待处理任务卡，不进入真实后台。

### 2. 计划确认页

目的：把目标转成结构化计划，由人确认后再执行。

必须包含：

- 原始目标回显。
- 6 步系统建议计划。
- 3 条验收标准。
- 风险提示。
- `确认并执行`。
- `保存草稿`。
- 右侧状态栏。
- 底部事件流。

关键交互：

- 点击 `确认并执行`，任务状态从待处理变为执行中，进入任务详情页。
- 点击 `保存草稿`，任务保持待处理，回到首页。
- 从待处理任务重新进入时，恢复到计划确认视图。

### 3. 任务板

目的：承载任务状态总览，是系统主轴视图。

任务板只使用五列：

- 待处理。
- 执行中。
- 验证中。
- 待审查。
- 已完成。

异常状态只作为标签展示，不作为看板列。

关键交互：

- 点击任务卡片，进入任务详情页。
- 待审查列任务可点击 `查看审查包`，进入审查包页。
- 不提供“运行”或“审查”一级导航。
- V0.1.1 不支持拖拽换列。

### 4. 任务详情页

目的：让用户查看单个任务的状态、上下文、验收标准和历史运行。

顶部按钮只保留：

- `查看运行详情`。
- `停止`。
- `重试`。
- `查看审查包`。

不要保留 `查看上下文` 按钮，因为上下文已经是任务详情页内部 Tab。

任务详情页 Tab：

- 概览。
- 上下文。
- 验收标准。
- 历史运行。

历史运行字段：

- 运行编号。
- 开始时间。
- 结束时间。
- 最终结果。
- 验证结果。
- 触发来源。
- `查看运行详情`。

关键交互：

- 点击 `查看运行详情`，进入执行详情页。
- 点击历史运行行，进入执行详情页并选中对应运行。
- 点击 `停止`，任务保留原主状态，叠加“已停止”标签。
- 点击 `重试`，新建一次运行，任务进入执行中。
- 点击 `查看审查包`，进入审查包页。

### 5. 执行详情页

目的：展示一次运行的完整证据链。

执行详情页只保留五个 Tab：

- 时间线。
- 上下文快照。
- 日志。
- 改动。
- 验证。

不要保留“产物”Tab。

改动 Tab 必须展示：

- 修改文件汇总。
- 修改文件列表。
- 至少一个静态差异样例。
- 改动摘要。

V0.1.1 不提供 `丢弃改动` 按钮。

验证未通过时，`查看审查包` 置灰，并提示：

> 验证未通过时无法查看审查包，请先重试或人工查看日志。

### 6. 审查包页

目的：给人做最终决策。它是 V0.1.1 最重要的最终决策入口。

必须包含：

- 任务回顾。
- 智能体总结。
- 验收标准结果。
- 修改文件清单。
- 同页静态差异预览。
- 验证结果。
- 风险提示。
- 审查建议。
- 底部固定决策区。

审查包页必须支持同页查看改动差异：

- 默认选中第一个文件。
- 点击文件行只更新同页差异预览。
- 不跳转到执行详情页。
- V0.1.1 至少展示一个静态 diff 样例。
- 不要求真实编辑能力。

底部固定决策区：

- 左侧：`拒绝并反馈`。
- 右侧：`接受结果`。
- 中间或次级位置：`更多操作`。

`更多操作` 中包含：

- `重新执行`。
- `让审查智能体复查`。

按钮语义：

| 操作 | 含义 | 是否需要反馈 | 状态变化 |
|---|---|---|---|
| 接受结果 | 用户认可本次结果 | 否 | 待审查 → 已完成 |
| 拒绝并反馈 | 用户不接受本次结果，并给出修改意见 | 是，必填 | 待审查 → 执行中，叠加“已拒绝”标签 |
| 重新执行 | 不带新反馈，按原计划重新跑一次 | 否 | 待审查 → 执行中，新建一次运行 |
| 让审查智能体复查 | 请求审查智能体再次检查本次结果 | 否 | 主状态保持待审查，右侧状态栏显示审查智能体工作中 |

接受结果后：

1. 弹出“是否保存为工作流？”提示。
2. 用户选择 `暂不保存`。
3. 任务进入已完成列。
4. 事件流出现“任务-001 已接受”。

## 状态机

主任务状态只保留五个：

```text
待处理 → 执行中 → 验证中 → 待审查 → 已完成
```

异常标签包括：

- 已停止。
- 执行失败。
- 验证未通过。
- 已拒绝。
- 卡住。
- 需要人工处理。

异常标签不作为任务板列名。

运行状态：

- 未开始。
- 进行中。
- 已完成。
- 已失败。
- 已停止。

验证状态：

- 未开始。
- 进行中。
- 已通过。
- 未通过。

智能体状态：

- 计划智能体：空闲 / 工作中。
- 工作智能体：空闲 / 工作中。
- 审查智能体：空闲 / 工作中。

完整状态流转与按钮状态变化见 `docs/03-状态机与跳转矩阵.md`。

## 样例数据

贯穿样例目标：

> 修复登录页错误提示不显示，并补充测试。

样例计划必须为 6 步：

1. 阅读登录页相关代码，定位错误提示组件。
2. 确认错误提示未显示的根因。
3. 修复错误提示显示逻辑。
4. 在登录页测试文件中补充错误提示相关用例。
5. 运行单元测试与组件测试。
6. 生成改动总结与审查建议。

样例验收标准必须为 3 条：

1. 输入错误密码后，登录页能展示对应错误提示。
2. 新增至少 2 条针对错误提示的测试用例，且全部通过。
3. 不引入新的代码检查或类型检查告警。

完整样例数据见 `docs/04-样例数据与事件文案.md`。

## 跳转矩阵

关键跳转：

- 首页 `生成任务计划` → 计划确认页。
- 计划确认页 `确认并执行` → 任务详情页，待处理变为执行中。
- 任务详情页 `查看运行详情` → 执行详情页。
- 执行详情页 `查看审查包` → 审查包页，验证通过后可用。
- 审查包页点击修改文件清单 → 审查包页同页差异预览，不跳转。
- 审查包页 `接受结果` → 任务板，待审查变为已完成。
- 审查包页 `拒绝并反馈` → 任务详情页，待审查变为执行中，并叠加“已拒绝”标签。
- 审查包页 `重新执行` → 任务详情页，待审查变为执行中，不带新反馈。
- 审查包页 `让审查智能体复查` → 审查包页，主状态保持待审查。

完整矩阵见 `docs/03-状态机与跳转矩阵.md`。

## 文案统一表

统一使用：

- 智能体，不用 Agent / 代理。
- 工作流，不用 Workflow。
- 审查包，不用 Review Packet。
- 接受结果，不用通过 / 合并。
- 拒绝并反馈，不用拒绝 / 驳回。
- 重新执行，不用重跑。
- 停止，不用中止 / 终止。

关键按钮：

| 统一文案 | 使用位置 | 含义 |
|---|---|---|
| 生成任务计划 | 首页 | 将目标拆解为计划 |
| 确认并执行 | 计划确认页 | 用户确认计划并启动任务 |
| 查看运行详情 | 任务详情页 | 进入执行详情页 |
| 查看审查包 | 任务板、任务详情页、执行详情页 | 进入审查包页 |
| 接受结果 | 审查包页 | 用户认可本次结果 |
| 拒绝并反馈 | 审查包页 | 用户拒绝本次结果并提交反馈 |
| 重新执行 | 审查包页更多操作 | 不带新反馈，按原计划重新跑一次 |
| 让审查智能体复查 | 审查包页更多操作 | 让审查智能体再次检查 |
| 停止 | 任务详情页 | 停止当前运行 |
| 重试 | 任务详情页 | 从失败、停止或卡住处重新发起一次运行 |
| 返回首页 | 占位页 | 回到首页 |
| 返回任务板 | 执行详情页、审查包页 | 回到任务板 |

- 审查包页底部固定决策区，左侧放 `拒绝并反馈`，右侧放 `接受结果`，其他操作放入 `更多操作`。

## 验收标准

V0.1.1 原型完成的判定条件：

1. 左侧导航只有首页、任务、工作流、智能体、设置。
2. 不出现“运行”“审查”一级导航。
3. 六个核心页面均可访问。
4. 工作流、智能体、设置三个占位页使用统一结构。
5. 样例流程可从首页走到接受结果。
6. 审查包页文件清单点击后同页展示差异。
7. 审查包底部按钮层级正确。
8. 右侧状态栏所有页面常驻。
9. 顶栏只展示“严格模式”小标签。
10. 任务状态只使用五个主状态。
11. 异常状态只作为标签展示。
12. 执行详情页不出现“产物”Tab。
13. 任务详情页不出现 `查看上下文` 按钮。
14. 验证未通过时 `查看审查包` 置灰，并显示指定提示。
15. 文案符合统一表。

Checklist 版本见 `docs/05-低保真原型验收清单.md`。
